3. 講義・技術内容
① Claude Codeとターミナル操作の優位性
なぜ重要か：ブラウザのチャットと違い、PCのファイルを直接操作できる「手足を持った相棒」。GoogleやExcelとの連携も同じスキルで実現できる汎用性がある。
  • ターミナル経由でファイルをダイレクトに作成・編集・実行できる
  • 将来的にGoogleドライブ・スプレッドシート・メールなどの自動化にも同じ知識が活きる
  • 住奥美穂氏がMCP（モデルコンテキストプロトコル）経由でGoogleドライブ接続に成功
② バージョン管理と復元方法
なぜ重要か：AIがコードを大きく変更する際に「戻せる」安心感があることで、大胆に試せるようになる。失敗を恐れない開発スタンスの土台。
  • コミット = セーブポイント、プッシュ = GitHubへの送り込み
  • Gitリバート（コミットIDを指定）：昨日以前の状態に戻したいとき
  • /rewindコマンド：チャット履歴内で直前の状態に戻したいとき
  • GitHubリポジトリは必ずプライベート設定で作成すること
③ UIコンポーネントの実装（ライブコーディング）
なぜ重要か：コンポーネントの名前を知るだけでAIへの指示精度が飛躍的に上がる。語彙がそのままAIへの指示力になる。
  • トースト通知：タスク追加時の完了メッセージ
  • プルダウン：カテゴリー選択（AIが自動でバッジも採用）
  • アコーディオン：タスク詳細の展開・折りたたみ
  • モーダル：タスク選択時の詳細表示画面
  • 確認ダイアログ：削除操作時の誤操作防止
  • バッジ：カテゴリーの視覚的区別（AIが自動で選択）
  • SVGによるイラスト生成：「SVGで書いて」と指示するだけで絵文字・アイコンを作成可能（石橋氏がもやし絵文字を実装）
④ Claude Codeの動作モード
なぜ重要か：モードを使い分けることで「考えさせる」「確認なしで進める」など開発効率を状況に応じて最適化できる。
  • ノーマルモード：通常の編集モード（毎回Yes確認あり）
  • プランモード：ファイルを編集せず仕様だけ検討するモード。複雑な機能追加前に使うと安全
  • バイパスパーミッションモード：毎回のYes確認を省略。作業が速くなるがリスクあり
  • Shift+Tabで3モードを切り替え
⑤ Anthropic API連携とAPIキーのセキュリティ管理
なぜ重要か：APIキーは「お金と同じ」。流出すると第三者に使われ課金が発生する。正しい管理方法を最初に身につけることが必須。
  • APIキー取得にはAnthropicコンソールでクレジットカード登録が必要（従量課金）
  • 絶対NG：APIキーをチャットや会話に直接貼り付けること（Claudeから警告が出る）
  • 正しい管理方法：プロジェクトルートに .env ファイルを作成し ANTHROPIC_API_KEY=xxxxx と記述
  • .gitignore に .env を記載することでGitHubへのアップロードを防ぐ
  • キー流出時は即座にAnthropicコンソールの3点メニューからキーを削除・無効化
⑥ AI自動タスクカテゴリー分類機能の実装
なぜ重要か：API連携の基本パターンを習得することで、AIを自分のアプリに「組み込む」感覚が身につく。これが各社の業務AIの基礎になる。
  • タスク登録時にAnthropicAPIにタスク内容を送信し、Claudeがカテゴリーを自動判定してアプリに登録
  • 内部プロンプト例：「あなたはタスク分類AIです。仕事・買い物・プライベート・健康の4カテゴリーのいずれかに分類してください」
  • 実演：「もやし買いに行く」→「買い物」と自動分類に成功
  • エラー発生時の対処：エラーメッセージをそのままClaude Codeに貼り付け → サーバー再起動で解決
  • ユーザーがプルダウンでカテゴリーを手動変更できる設計も実装
⑦ コンテキストエンジニアリング
なぜ重要か：AIへの指示の「質」が出力の質を直接決める。コンテキストを意識するだけで同じAIから全く違うアウトプットが得られる。
  • /clear コマンド：別タスクに移る前にAIの記憶（コンテキスト）をリセット
  • AIに役割を与える（「あなたはプロのUI設計者です」「品質チェックエージェントとして動いてください」）と出力品質が向上
  • 技術スタック（Next.js・TypeScriptなど）をアプリ開発の最初に伝えることでブレが減る
4. 主要ディスカッション（Q&A）
Q.ターミナルでClaude Codeを使うことのメリットは何か？ブラウザのチャットと何が違うのか？
ブラウザのチャットは「会話だけ」だが、Claude Codeは実際にファイルを作成・編集・実行できる。PCに「手足が生えた状態」になるイメージ。将来的にGoogleカレンダーやスプレッドシートとの連携も同じ知識ベースで実現できるため、学んだスキルが他の自動化にも転用できる汎用スキルになる。
Q. エラーが出たときの対処法は？
エラーメッセージをそのままClaude Codeのチャットに貼り付けるだけでよい。今回のセッションでもAPI連携実装直後にエラーが発生したが、エラーメッセージを貼り付け → サーバー再起動の指示で数分以内に解決した。エラーを「ラッキーな学習機会」として捉える姿勢が大切。
Q. Google連携にはどんな方法があるか？
3つの方式がある。①サービスアカウント方式（バックエンドからサーバー間で接続）、②OAuth方式（ユーザーがログインして許可する方式）、③MCP（モデルコンテキストプロトコル）方式（Claude Codeから直接Googleサービスに接続）。

6. 豆知識 Tips
⌨️
Shift+Tab で3モード即切り替え
Claude Code は Shift+Tab を押すたびに「ノーマル → プラン → バイパス」の3モードを順番に切り替えられる。複雑な機能を追加するときはプランモードで安全確認、作業が慣れてきたらバイパスで爆速モードに。
🧹
/clear でAIの頭をリフレッシュ
Claude Code は会話履歴を「コンテキスト」として保持し続ける。別のタスクに移るとき /clear を打つと記憶がリセットされ、前の作業の影響を受けない新鮮な状態で指示できる。
🎭
AIに「役割」を与えると出力が変わる
「あなたはプロのUI設計者です」「品質チェックエージェントとして動いてください」のように役割を最初に宣言するだけで、同じ質問でも出力の質が大きく変わる。
🎨
「SVGで書いて」でオリジナルアイコンを作れる
画像ファイルを用意しなくても、「SVGでもやしのイラストを書いて」と指示するだけでコード上にアイコンやイラストを生成できる。フォントや絵文字に縛られない表現が可能になる。
🚨
エラーはそのままコピペするだけでOK
エラーが出たら内容を理解しようとしなくていい。ターミナルのエラーメッセージをそのままClaude Codeのチャットに貼り付けるだけで、原因の特定から修正まで自動でやってくれる。
🔑
APIキーは「お金と同じ」扱いで
APIキーが流出すると第三者に勝手に使われ課金が発生する。チャットや会話に貼らず、必ず .env ファイルに保存。もし流出したらAnthropicコンソールの3点メニューから即削除・無効化を。
💡
アプリの仕様はAIに提案させる
「何を作ればいいかわからない」ときは、自分の業務内容や課題をAIに伝えるだけでOK。「どんな機能があると便利か提案して」と聞けば、画面構成・必要なデータ・操作フローまでまとめてくれる。ゼロから考えるより圧倒的に速い。
