4. 講義・技術内容
① バイブコーディング（Vibe Coding）とは
なぜ重要か：日本語でAIに指示するだけでアプリが作れる時代が来た。非エンジニアが開発できる「デジタルの民主化」の核心。
  • 「雰囲気でコーディングをしていく手法」であり、生成AIが最も得意とする分野
  • Claude AIでオセロゲームを約30秒で作成し、日本語指示で石の色を変更するデモを実施
  • 外部ベンダーに依頼すると1,000万円を超えるシステムも、AIを使えば自然言語で構築可能
  • 意識すべき4つの概念：まずAI / AIの答えを選択する力 / データベースの主権 / AIレディ
② 内製化の意義と統合プラットフォーム
なぜ重要か：外部SaaSに依存するとデータ活用に制限がかかる。自社DBをコントロール下に置くことが、AI活用推進における最大のメリット。
  • 認証機能を備えた統合プラットフォーム「OKADA Board」のデモを実施
  • SaaSと異なり、アカウント数に関わらずコストゼロで社内利用できる
  • 手順書の品質評価をAIが行うなど、業務に特化したAI機能をピンポイントで実装できる
  • 現場からの要望をすぐに反映できる内製化は、外部発注では実現できない即応性を持つ
③ Webアプリの三層構造
なぜ重要か：構造を言語化できるようになることで、AIに正確な指示が出せるようになる。
  • フロントエンド：ユーザーが見る画面（HTML・CSS・JavaScript）
  • バックエンド：フロントエンドからの指示を受け、データを処理してDBと連携
  • データベース：データの格納場所 ※主権を自社で握ることが最重要
  • 使用技術：Next.js（フロント・バックを一括処理できる主流フレームワーク）＋ TypeScript
  • 「Next.jsとTypeScriptを使って」と指示できれば、細部の知識は不要
④ Claude Code の環境構築とモデル選択
なぜ重要か：ターミナル操作はPCのファイルを直接編集できる「手足が生えた状態」。将来のAIエージェント構築にも最も主流の手法。
  • ターミナルで「claude」と入力して起動
  • モデル選択：Opus（高性能）/ Sonnet（推奨・コーディングに最適）/ Haiku（高速・軽量）
  • Opusは制限が早く来る可能性があるため、通常はSonnetを使用
  • フォルダのパスをCDコマンドで指定してからClaude Codeを起動すること
  • ローカルフォルダはOneDriveなどクラウド上ではなく、ローカルディスクに置くことを推奨
⑤ Next.js でToDoアプリ作成
なぜ重要か：ToDoアプリにはUI・DB・AI・デプロイなどWebアプリに必要な要素が凝縮されており、基礎概念を効率よく習得できる。
  • 「Next.jsでToDoアプリのフロントエンドをDBなしでシンプルに作成して」とAIに指示
  • npm run dev でローカルサーバー起動 → localhost:3000 で動作確認
  • 「もやし」などのタスクを追加し動作確認。ページ更新でデータが消える→次回DBを接続
  • AIの挙動を監視し、意図しない場所へのフォルダ作成などはESCキーで強制停止
  • フォルダ名・ファイル名は半角スペースや大文字を避け、すべて小文字推奨
⑥ GitHub連携とバージョン管理
なぜ重要か：ソースコードの安全な管理と共有に不可欠。Vercelデプロイの基盤にもなる。
  • プッシュ（Push）：作業環境からGitHubのリポジトリにデータを送り込む
  • プル（Pull）：GitHubリポジトリからデータを取り出す
  • リポジトリは必ずPrivate設定で作成すること（ソースコード流出防止）
  • 参加者全員がGitHub認証・メール認証を完了し、ToDoアプリをリポジトリへプッシュ
5. 主要ディスカッション（Q&A）
Q. 内製化とSaaSサービスを使う場合のコスト差
外部ベンダーへの発注では1,000万円を超えることもあるシステムでも、AIを活用した内製化では大幅にコストを削減できる。さらにSaaSと異なり、ユーザー数に関わらず追加料金がかからず、現場の要望をすぐ反映できる点も大きなメリット。
Q. エラーが出た場合はどう対処すればよいか
エラーはラッキーだと思う姿勢が重要。エラーログをそのままAIに投げれば解決策を提示してくれる。このトラブルシューティング力こそがAI活用の鍵であり、AIとの対話量を増やすことで鍛えられる。
Q. AIが意図しない場所にフォルダを作ってしまった場合
ESCキーを押して強制終了し、フォルダ名・場所を修正してから再実行する。AIが勝手に作業を進める場合があるため、挙動を監視して問題があれば停止させる力を身につけることが大切。
Q. データベースの主権確保が重要な理由
外部SaaSにデータを預けると、将来のAI活用がベンダーの仕様や追加料金に縛られる。今から「AIレディ」な状態（良質なデータを自社DBとして保有）を構築しておくことが、中長期的な競争優位に直結する。
