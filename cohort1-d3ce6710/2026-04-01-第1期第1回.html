4. 講義・技術内容
（1）バイブコーディングとは
💡 なぜ重要か：これがこの講座の核心。「AIに頼む」ではなく「AIと協働する」感覚を最初に掴むことが大切です。
  • 自然言語（日本語）でAIに要望を伝えてコードを生成させる手法。2024年頃から普及
  • 非エンジニアでも開発可能にする「デジタルの民主化」の時代が到来
  • 実演：Claude AIで日本語指示のみ → 約1分でオセロゲーム（動作するWebアプリ）を生成
（2）バイブコーディングの2大ポイント
ポイント内容
① 概念理解 WebアプリがどのようなしくみForで動いているかを理解し、適切な用語でAIに指示を出す力。三層構造（フロントエンド・バックエンド・データベース）の把握が最重要
② トラブルシューティング力 AIとの対話量に比例して向上。エラーをそのままAIに貼り付けて解決策を得る。「AIに聞く → AIが出した選択肢から選ぶ」という思考シフトが重要
（3）システム内製化の重要ポイント
💡 なぜ重要か：今から自社DBにデータを溜めておくことが、将来のAI活用の土台になります。外部SaaSに頼りすぎると、その恩恵を受けられなくなります。
  • 【データベース主権】：外部SaaSにデータを預けると、将来のAI活用が外部ベンダー依存・追加料金になる
  • 【AIレディ状態】：今から自社DBに質の高いデータを蓄積しておくことで、AI活用の土台を先行整備
  • 【AIの組み込み方】：汎用AIチャットを現場に渡すのではなく、内製アプリにAIをピンポイントで差し込む手法が中小企業の唯一の方法
  • 【統合プラットフォーム構想】：ログイン画面を起点に、各種業務アプリを搭載した社内統合基盤を構築
（4）Webアプリの三層構造（概念）
層役割技術・言語
フロントエンドユーザーが見る画面HTML（構造）・CSS（デザイン）・JavaScript（動き）
バックエンドリクエスト処理・DB連携・外部API連携Next.js（フロント・バック共通フレームワーク）
データベースデータの永続保存場所次回以降接続予定。Neon等のクラウドDB
（5）開発ツール選定：Claude Codeターミナル方式を採用
  • 当初予定のReplitよりも、現在最注目のClaude Codeターミナル操作方式を採用（上級者向けだが最も処理速度が速い）
  • Claude Codeのモデル：Opus 4.6（高性能・高価格）/ Sonnet 4.6（推奨・中間）/ Haiku（安価・高速）
  • ターミナル方式はVS Code拡張機能方式より処理速度が高速。将来のAIエージェント構築にも最も適した手法
（6）ToDoアプリ作成〜GitHub連携（ハンズオン）
  • 新規プロジェクトフォルダーを作成し、ターミナルでClaude Codeにフォルダーを指定（CDコマンド）
  • 「Next.jsでToDoアプリのフロントエンドを最低限の仕様で作成してください」と日本語で指示
  • npm run dev コマンドでローカルサーバー起動 → ブラウザの localhost:3000 でプレビュー確認
  • GitHubにリポジトリを作成しコードをプッシュ。コミット（セーブ）→ プッシュ（倉庫に送る）の2段構造を習得
  • 【重要】リポジトリはセキュリティのため必ずPrivate設定にすること
