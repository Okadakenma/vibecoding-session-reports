4. 講義・技術内容
① バイブコーディング（Vibe Coding）とは
なぜ重要か：日本語でAIに指示するだけでアプリが作れる時代が来た。非エンジニアが開発できる「デジタルの民主化」の核心。
  • 自然言語（日本語）でAIに要望を伝え、コードを生成させる手法
  • Claude AIでオセロゲームを約1分で作成するデモを実施
  • 2大ポイント：①概念を理解してAIに適切な指示を出す、②トラブルシューティング力（AIとの対話量で鍛える）
② Webアプリの三層構造
なぜ重要か：構造を理解することで、AIに正確な指示が出せるようになる。
  • フロントエンド：ユーザーが見る画面（HTML・CSS・JavaScript）
  • バックエンド：リクエスト処理・データベースアクセス・外部連携
  • データベース：データの格納場所 ※主権を自社で握ることが最重要
③ Claude Code とターミナル操作
なぜ重要か：ターミナルはPCのファイルを直接編集できる「手足が生えた状態」。Webブラウザチャットより処理速度が速く、将来のAIエージェント作成にも最も主流の手法。
  • モデル：Opus 4.6（高性能）/ Sonnet 4.6（推奨）/ Haiku（高速・安価）
  • ターミナルで「claude」と入力して起動
  • Claude Code Pro プランへの切り替えを全員確認
④ Next.js でToDoアプリ作成
なぜ重要か：Next.jsはフロントエンド・バックエンドを一つのフレームワークで構築できる現在の主流技術。
  • 新しいプロジェクトフォルダーを作成し、ターミナルで指定
  • 「Next.jsでToDoアプリのフロントエンドを最低限の仕様で作成して」とAIに指示
  • npm run dev でローカルサーバー起動 → localhost:3000 で動作確認
⑤ GitHub連携
なぜ重要か：ソースコードの安全な管理と共有に不可欠。チーム開発・Vercelデプロイの基盤。
  • コミット：ゲームでいう「セーブ」
  • プッシュ：セーブしたものをGitHub倉庫に送る
  • プル：倉庫からデータを取り出す
  • リポジトリは必ずプライベート設定で作成すること（セキュリティ上の重要事項）
5. 主要ディスカッション（Q&A）
Q. AIへの指示の口調によって結果が変わるのか？（田中かける）
口調を変えることでAIが「ちゃんとやったりする」ことがある。AIはユーザーの感情を読み取り、例えば同じミスを繰り返す場合は別のアプローチを提案するなど、対話のトーンに応じた動作変化がある。
Q. 教材はすべて岡田さんが作ったのか、AIが関与しているのか？（文田康祐）
AIと協力しながら要素を決め、インターネットページを作成している。Webページ作成はAIが最も得意とする分野。教材の集中管理により受講生にとって分かりやすい形を目指している。
Q. コマンドプロンプトを使う方法は初心者に難しすぎないか？（文田康祐）
今回の方法は最も上級者向けだが、「これができれば、より簡単な方法もすべて扱えるようになる」という考えから採用。将来のWebアプリ作成後のAIエージェント構築において最も主流となる手法であるため、あえて難易度の高い方法を選択した。
Q. データベースの主権確保が重要な理由は？（各受講生）
データベースを外部に委ねると、将来のAI活用が外部ベンダーのプラットフォームや追加料金に依存することになる。今から「AIレディ」な状態（質の高いデータセットを自社DBとして保有）を構築しておくことが中長期的な競争優位につながる。
