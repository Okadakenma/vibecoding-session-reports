Vercelデプロイ手順ガイド（GitHub連携）

GitHubと連携して自動デプロイ環境を構築するステップバイステップガイド
デプロイ手順
STEP 0：Vercel CLIのインストール任意
  • ターミナルで以下を実行してVercel CLIをインストールする
npm install -g vercel
  • インストール後、以下でVercelにログイン（ブラウザが開いて認証される）
vercel login
CLIは任意。GitHub連携によるWeb画面からのデプロイのみであれば不要。
STEP 1：Vercelアカウント作成・GitHubで認証
GitHubアカウントで認証することが自動デプロイの前提条件。メール登録の場合は後からGitHub連携が必要になる。
  • vercel.com にアクセスする
  • 「Sign Up」→「Continue with GitHub」を選択する
  • GitHubの認証画面でアカウントを選択して許可する
  • Vercelのダッシュボードが表示されればアカウント作成完了
STEP 2：VercelにGitHub Appをインストール
VercelがGitHubリポジトリを読み取るために、GitHub App（連携アプリ）のインストールが必要。初回のみ実施。
  • 「Add New → Project」をクリックすると、GitHubリポジトリの選択画面が表示される
  • リポジトリが表示されない場合は「Adjust GitHub App Permissions」または「Install GitHub App」をクリック
  • GitHubのインストール画面で「All repositories（全リポジトリ）」または「Only select repositories（特定リポジトリのみ）」を選択
  • 「Install」ボタンをクリックして許可する
  • Vercelの画面に戻ると、リポジトリ一覧が表示される
STEP 3：GitHubにコードをプッシュ
  • デプロイしたいコードをGitHubにプッシュする
# 変更をすべてステージング git add . # コミットメッセージを付けて確定 git commit -m "デプロイ準備" # GitHubのmainブランチにプッシュ git push origin main
STEP 4：Vercelで新しいプロジェクトを作成
  • Vercelダッシュボードで「Add New → Project」をクリック
  • 「Import Git Repository」の一覧からデプロイしたいリポジトリを選択
  • 「Import」をクリック
  • Framework Preset が「Next.js」になっていることを確認する（自動検出される）
STEP 5：環境変数を設定（重要）
.env.local は .gitignore によりGitHubに上がらない。Vercel側で別途設定しないとデプロイ後にDB接続が失敗するため必須。
  • プロジェクト作成画面の「Environment Variables」セクションを開く
  • 以下の変数を1つずつ「Name」「Value」に入力して「Add」をクリック
# データベース接続URL（Neonのコネクション文字列） DATABASE_URL = postgresql://... # Anthropic APIキー ANTHROPIC_API_KEY = sk-ant-... # その他 .env.local に記載の変数をすべて追加
後から追加・変更する場合は、Vercelダッシュボードの「Settings → Environment Variables」から編集できる。変更後は再デプロイが必要。
STEP 6：デプロイ実行
  • 「Deploy」ボタンをクリックする
  • ビルドログがリアルタイムで表示される（数分かかる）
  • 「Congratulations!」と表示されればデプロイ成功
  • 発行されたURLをクリックしてアプリが表示されることを確認する
STEP 7：以降は自動デプロイ
  • GitHubの main ブランチに git push するたびに、Vercelが自動でビルド・デプロイを実行する
  • デプロイ状況はVercelダッシュボードの「Deployments」タブで確認できる
コードを修正したら「git add → git commit → git push」の3ステップだけで本番環境に反映される。
エラー時の対処法
ビルドエラーの場合
  • Vercelダッシュボードの「Deployments」タブを開く
  • 失敗したデプロイをクリック →「View Build Logs」を開く
  • 赤く表示されているエラーログをコピーして、Claude Codeに貼り付ける
デプロイ成功後にアプリがエラーになる場合
  • まず環境変数の設定漏れを確認する（「Settings → Environment Variables」）
  • ブラウザで Ctrl+Shift+I（Mac: Cmd+Option+I）を押してコンソールを開く
  • 赤いエラーメッセージをコピーしてClaude Codeに貼り付ける
DB接続エラーが多い場合は DATABASE_URL の値が正しく設定されているか確認する。Neonのダッシュボードからコネクション文字列をコピーし直すと解決することが多い。
