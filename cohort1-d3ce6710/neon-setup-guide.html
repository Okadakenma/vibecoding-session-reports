Neonデータベース セットアップガイド

アカウント作成からコネクション文字列の取得・アプリへの設定まで
セットアップ手順
STEP 1：Neonアカウントを作成する
Neonはクラウド上のPostgreSQLデータベースサービス。ローカルにデータを保存する代わりにNeonを使うことで、どこからでもアクセスできる本格的なDBが無料で使える。
  • neon.tech にアクセスする
  • 「Sign Up」をクリック → 「Continue with GitHub」でGitHubアカウントで登録する
  • メール認証が求められた場合は届いたメールのリンクをクリックする
  • ダッシュボードが表示されればアカウント作成完了
STEP 2：新しいプロジェクトを作成する
  • Neonダッシュボードで「New Project」をクリック
  • 以下を設定する：
    • Project name：アプリ名を入力（例：todo-app）
    • Database name：そのままでOK（デフォルト: neondb）
    • Region：Asia Pacific (Tokyo) を選択（日本から最速）
  • 「Create Project」をクリック
無料プランで1プロジェクト・0.5GBまで利用可能。学習用途には十分な容量。
STEP 3：コネクション文字列を取得する
コネクション文字列はアプリがデータベースに接続するためのURL（パスワード相当）。これを取得してアプリの設定ファイルに貼り付けることでDB接続が確立する。
  • プロジェクト作成後、ダッシュボードの「Connection Details」または「Connect」ボタンをクリック
  • 表示される接続情報の中から 「Connection string」 のタブを選択
  • 以下のような文字列が表示されるのでコピーボタンを押してコピーする
# コネクション文字列の例（postgresqlで始まる長いURL） postgresql://username:password@ep-xxx.ap-southeast-1.aws.neon.tech/neondb?sslmode=require
コネクション文字列にはパスワードが含まれる。GitHubには絶対にアップロードしないこと（.gitignoreで保護されているが注意）。
STEP 4：.env.local に登録する
  • プロジェクトのルートにある .env.local ファイルをVS Codeで開く
  • 以下の行を追加して、コピーしたコネクション文字列を貼り付ける
DATABASE_URL=postgresql://username:password@ep-xxx.ap-southeast-1.aws.neon.tech/neondb?sslmode=require
  • ファイルを保存する（Cmd+S / Ctrl+S）
.env.local ファイルが存在しない場合は、プロジェクトのルートに新規作成する。
STEP 5：Claude Codeでテーブルを自動生成する
  • .env.local を保存後、Claude Codeに以下のように指示する
# Claude Codeへの指示例 「DATABASE_URLを設定しました。Prismaを使ってデータベースにテーブルを作成してください」
  • Claude Codeが prisma migrate dev を実行し、テーブルが自動生成される
  • Neonダッシュボードの「Tables」タブでテーブルが作成されたことを確認できる
コネクション文字列を再取得する場合
新しいプロジェクトに切り替えた・文字列を紛失した場合
  • Neonダッシュボードにログインする
  • 対象のプロジェクトを選択する
  • 左メニューの「Connection Details」をクリック
  • 「Connection string」をコピーして .env.local の DATABASE_URL を更新する
  • Vercelにもデプロイしている場合は「Settings → Environment Variables」の DATABASE_URL も同様に更新し、再デプロイする
.env と .env.local の両方にDATABASE_URLが存在する場合、.env.local が優先される。古い値が残っていると競合するため、両ファイルを確認すること。
